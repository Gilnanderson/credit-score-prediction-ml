# 📊 Credit Score Prediction with Machine Learning

## 🎯 Project Objective
The objective of this project is to build a Machine Learning model capable of analysing the financial profile of clients and automatically predicting their **Credit Score**. The system analyses a historical database to classify new clients, automating and bringing mathematical precision to credit risk analysis.

## 🛠️ Technologies and Libraries Used
* **Python** (Main language)
* **Pandas** (For data manipulation, cleaning, and formatting)
* **Scikit-Learn** (For data pre-processing, training, and evaluation metrics)
* **Matplotlib** (For data visualisation and chart generation)

## 🧠 Machine Learning Models
To ensure the best result, we trained and compared two different algorithms:
1. **Random Forest Classifier**: A robust ensemble model that creates multiple decision trees to make highly accurate predictions, handling complex interactions between variables very well.
2. **KNN (K-Nearest Neighbours)**: A model that classifies data based on the proximity and similarity of financial behaviour among clients (the "nearest neighbours").

## ⚙️ Step-by-Step Development
1. **Exploratory Data Analysis (EDA):** Importing the `clientes.csv` database and conducting an initial analysis of statistics and column types.
2. **Pre-processing:** * Identifying text columns (categorical) and converting them into numerical data using `LabelEncoder`.
   * Removing columns that are irrelevant to the model, such as `id_cliente`.
3. **Train and Test Split:** Splitting the database, allocating 70% for training the Artificial Intelligence and 30% for testing its effectiveness on unseen data.
4. **Baseline Calculation:** Analysing the statistical probability if the model simply guessed the most common answer ("Standard"), providing a real baseline for comparison.
5. **Evaluation and Metrics:** Calculating the accuracy of both models using the `accuracy_score` metric and plotting a comparative bar chart.
6. **Feature Importance:** Mathematical extraction of the weight of each variable in the Random Forest's decision-making process, displayed both in a table and in a horizontal bar chart.
7. **Final Presentation:** Customised formatting of the predictions table, converting numerical financial values into the Brazilian currency format (**R$ 0.000,00**) natively.

## 📈 Key Insights
* The model evaluates dozens of characteristics, but the **Feature Importance** analysis generated by the code visually revealed which client attributes (such as total debt and interest rates) carry the greatest weight in the final definition of their score.
* A clear visualisation was generated to compare the performance (accuracy) of the Random Forest against the KNN, justifying the choice of the final model.
